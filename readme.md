# Medishare, l'application de partage de données entre médecin et patient:
<img width="1103" height="627" alt="image" src="https://github.com/user-attachments/assets/701edff9-3e16-431f-9452-80a9b1a233d6" />

Ce projet consiste à concevoir et déployer une plateforme de
gestion de contenus sécurisée dans le domaine de la santé (prototype). Il s’agit d’un site où les patients et les médecins peuvent consulter leurs documents partagés. Cette idée nous est venu après s’être rendu compte que certains rendez-vous médicaux planifiés plusieurs mois à l’avance pouvaient nous contraindre à perdre des documents papiers. Ainsi , une solution digitale ferait ce problème d’ordonnances/radios perdues.

Cette plateforme permet à un patient , avec un dossier unique, de partager à son médecin attitré des documents. Le médecin attitré se connecte alors sur la plateforme et peut consulter les fichiers du patient.
Le projet intègre des pratiques DevSecOps, utilise les services AWS pour
l’infrastructure cloud, et inclut une recherche sur les attaques adverses en cybersécurité
pertinentes au contexte de la gestion de contenus numériques.

#Objectifs pédagogiques :

- Concevoir et déployer une application web sécurisée pour la gestion et le partage de
contenus numériques.
- Intégrer des pratiques DevSecOps pour assurer la sécurité continue tout au long du
cycle de développement.
- Utiliser les services AWS pour l’hébergement, le stockage, la gestion des clés et la
surveillance de l’application.
- Réaliser une recherche sur les attaques adverses spécifiques aux plateformes de
gestion de contenus numériques.
- Conceptualiser l’architecture cloud de la plateforme en utilisant Cloudcraft.

# Conclusion
Ce projet de cloud security sur aws fut très enrichissant et nous avons pu apprendre à mettre en pratique conjointement un grand nombre de services. Le tout nous a permis de comprendre l’intérêt d’une infrastructure as code grâce à nos fichiers terraform. Nous avons pu comprendre toute la puissance du cloud AWS lorsqu’il s’agit de haute disponibilité (multi-az, auto scaling group,...) et de sécurité (roles,acl,waf,kms,...).
